# 常见报错

### 1. Property "About" was accessed during render but is not defined on instance.

- 指About没在data或props中定义



### 2. "Unexpected mutation of 'info' prop vue/no-mutating-props" 时

- 这通常意味着你试图修改了一个传递给子组件的 prop（属性）违背了单向数据流原则。



### 3.  warning报错:值要带''双引号 

 <button @click="currentPage = 'About'">about</button>

<button @click="currentPage = 'Home'">home</button>



### 4.'defineProps' is not defined (no-undef)

- 这个错误说明 defineProps 没有被识别为全局 API。这通常发生在 ESLint 规则中没有正确配置 Vue 3 的组合式 API (Composition API) 或 <script setup> 特性

解决:

- npm install eslint-plugin-vue@latest --save-dev

- npm install vue-eslint-parser --save-dev

- vue-eslint-parser 和 eslint-plugin-vue 扮演不同的角色，且两者是互补的，因此即使你更新了 eslint-plugin-vue 到最新版本





### 5. 报错时:__VUE_PROD_HYDRATION_MISMATCH_DETAILS__ is not explicitly

-  cli的vue.config.js(config配置)复制粘贴

```javascript
const { defineConfig } = require('@vue/cli-service')
const webpack = require('webpack');
module.exports = defineConfig({
 transpileDependencies: true,
 configureWebpack: {
  resolve:{
   alias:{
​    "a": "@/a",//自己配置,但会被jsconfig.json中配置覆盖,同时没有代码提示,使用推荐去jsconfig.json中配置(也就是方法一)
​    "Components": "@/Components"
   }
  },
  plugins: [
   new webpack.DefinePlugin({
​    '__VUE_PROD_HYDRATION_MISMATCH_DETAILS__': true,

​    // 其他特性标志

   })
  ]
 }
})
```







### 6.Component name "About" should always be multi-word  vue/multi-word-component-names

- solve:

在eslintrc.js 复制粘贴或在其rules中添加'vue/multi-word-component-names': 'off',如下图所示

```javascript
module.exports = {
 root: true,
 env: {
  node: true
 },
 'extends': [
  'plugin:vue/vue3-essential',
  'eslint:recommended'
 ],
 parserOptions: {
  parser: '@babel/eslint-parser'
 },
 rules: {
  'no-console': process.env.NODE_ENV === 'production' ? 'warn' : 'off',
  'no-debugger': process.env.NODE_ENV === 'production' ? 'warn' : 'off',
  'vue/multi-word-component-names': 'off'
 }
}
```
